# Memory Jar 🧠

> 你的私人记忆罐子 - 存入一切，随时召回

---

## 📌 项目概述

### 一句话描述
一个个人知识库问答系统，用户存入文档后，可以用自然语言提问，AI 基于文档内容回答。

### 核心价值
- 你记不住的，它帮你记住
- 你想问的，它基于你的数据回答
- 你的知识，永远在你手里
- **私人模式**：存储生活、学习、工作经验，随时找回珍贵记忆

### 技术栈
| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 19 + TypeScript + TailwindCSS | 聊天界面 + 文档管理 |
| 后端 | FastAPI + LangChain | API 服务 + RAG 检索 |
| 数据库 | SQLite | 用户、文档、对话记录 |
| 向量库 | Chroma (或本地 pickle) | 文档向量存储 |
| LLM | DeepSeek API | 问答生成 |

## 🎯 功能范围

### 私人模式（私密问答）
专为存储和查询个人隐私信息设计，数据完全本地存储，确保隐私安全：

**私密信息查询示例**：
- 车位管理费到几月到期？
- 本月还有哪些未完成的制定好的事情？
- 我的羽毛球寄到哪里了？
- 去年体检报告的关键指标是什么？
- 家里备用钥匙放在哪里？

**核心特点**：
- 存储个人敏感信息（财务、健康、家庭事务等）
- 自然语言查询，快速定位所需信息
- 数据本地存储，无云端同步风险

### 典型使用场景（日常问答）
聊天支持两类问题：**优先**基于已存文档检索回答；若无相关文档或属于通用问题，也可进行日常知识问答。例如：

**日常问答示例**：
- 什么是 RAG 技术？
- 如何配置 FastAPI 跨域？
- 《百年孤独》的主要人物有哪些？
- Python 装饰器的使用方法？
- 上次会议讨论的重点是什么？

**适用场景**：
- 学习笔记管理：整理课程笔记、知识点总结
- 文档问答：基于上传的文档进行智能问答
- 知识回顾：快速复习已存储的知识内容
- 工作辅助：项目文档查询、技术方案检索

## 📤 文档上传（V1.0）

用户**无需填写表单**，直接选择本地文件上传。后端自动用**文件名**作为 `title`，读取文件内容写入 `content`。

### 接口

```
POST /api/documents/upload/
Content-Type: multipart/form-data
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `file` | File | 是 | 上传的文件 |

**支持扩展名**：`.txt` `.md` `.pdf` `.doc` `.docx` `.json` `.xml` `.csv` `.xlsx`

**响应示例**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "title": "物业缴费记录.md"
  }
}
```

### 前端 / 原型交互

**对话页文案**（避免歧义：不仅可查存入记忆，也支持日常问答）：
- 副标题：「优先检索你的文档，也支持日常问答」
- 输入框占位：「问待办、笔记内容，或任何日常问题…」
- 加载中：「正在检索并生成回答…」

- 文档页点击「上传文档」或拖拽区域，**只选文件**，不出现标题、内容输入框
- 上传中显示进度或 loading；成功后刷新文档列表
- 列表展示 `title`（来自文件名）与上传时间

## 🗄️ 数据库设计

### SQLite 表结构

```sql
-- 文档表
CREATE TABLE documents (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 对话表
CREATE TABLE conversations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    question TEXT NOT NULL,
    answer TEXT NOT NULL,
    sources TEXT,  -- JSON 格式，存储引用来源
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
